# 健康指标异常判断：相对值 vs 绝对值 vs 混合策略

## 背景
PeakWatch 产品需要对用户的身体指标（静息心率、HRV、呼吸节律、血氧饱和度、手腕温度）进行异常值判断和预警。

现有系统主要使用相对值判断（±1 标准差），基于用户个人基线进行偏离检测。但这存在盲区：例如静息心率持续高于 100 bpm 即使在个人基线范围内，医学上也属于异常（心动过速）。

## 冲突点
**个体化精准监测** vs **医学安全边界**

- 只用相对值：能捕捉个体变化，但可能忽略绝对危险值
- 只用绝对值：符合医学标准，但无法识别"对这个人来说的异常"
- 个体差异很大（如运动员静息心率可能 40-50 bpm），固定阈值会误报

## 具体例子
某用户的静息心率场景：

- 方案 A（纯相对值）：用户基线 95 bpm，标准差 5，异常阈值 90-100 bpm → 当心率升至 105 bpm 时触发预警，但 98 bpm（虽接近心动过速）不报警
- 方案 B（纯绝对值）：固定阈值 50-100 bpm → 运动员用户心率 45 bpm 被误报为异常（实际正常）
- 方案 C（混合策略）：±1σ **且** 50-100 bpm → 既能捕捉个体异常变化，又能拦截医学危险值

## 我的决策
**采用方案 C：混合策略 - 相对值 + 绝对值双重边界**

具体规则设计：
```
静息心率：±1σ 且 50-100 bpm
HRV：±1σ 且 SDNN >20 ms
呼吸节律：±1σ 且 12-20 次/分
血氧饱和度：±1σ 且 ≥95%（<90%立即预警）
手腕温度：±1℃/1.8℉ 且 30-37℃
```

**判断逻辑：**
- 满足相对值**或**绝对值异常 → 触发预警
- 绝对值边界为硬阈值，超出必然异常（不可覆盖）
- 相对值边界捕捉个体基线变化（更敏感）

**理由：**
1. 医学安全优先：绝对值边界防止漏报危险情况
2. 个体化监测：相对值捕捉"对你来说的异常"
3. 减少误报：运动员等特殊群体不会因为基线不同被误判
4. 用户体验：既有通用医学参考，又有个性化监测

## 决策时的担忧
1. **双重边界是否会增加误报？**
   - 待验证：上线后监控假阳性率（目标 <5%）
   - 缓解策略：设置"关注"和"警告"两级阈值

2. **相对值标准差计算是否稳定？**
   - 担心：新用户数据不足时标准差不准
   - 方案：前 30 天数据积累期使用纯绝对值，之后切换混合模式

3. **绝对值边界是否需要分人群？**
   - 运动员、老年人的"正常范围"可能不同
   - 待优化：未来考虑按年龄/运动水平分层设置

4. **温度指标环境因素影响大**
   - 手腕温度受环境温度影响，绝对值边界可能不够准确
   - 待验证：收集三个月数据，分析环境温度关联性

## 决策原则提炼
> **个体化健康监测需要"相对变化"和"绝对安全"双重保障：用相对值捕捉个人异常趋势，用绝对值守住医学底线，任一触发即预警**
