# DecisionLog Skill

提取对话中的决策模式并保存为结构化 Markdown 文件。

## 功能

- 从对话中自动提取决策要素（背景、冲突、方案、结果）
- 生成统一格式的决策记录文档
- 沉淀可复用的决策原则

## 使用方法

### 触发 Skill

在对话中说：
- "帮我总结这个决策"
- "记录一下这个决定"
- "抽象一下这个问题"
- "形成决策案例"

或者直接调用：
```bash
/decision-log
```

### 输出位置

决策记录保存在：
```
claude-workspace/decisions/YYYY-MM-DD-{short-title}.md
```

## 文档格式

每个决策记录包含：

```markdown
# {案例标题}

## 背景
决策的上下文和触发因素

## 冲突点
核心的权衡和矛盾

## 具体例子
说明问题的具体场景

- 方案 A（标签）：描述 → 后果
- 方案 B（标签）：描述 → 后果
- 方案 C（标签）：描述 → 后果

## 我的决策
最终决定和理由

## 决策时的担忧
当时的顾虑（后续可验证）

## 决策原则提炼
> **从这个决策中提取的一句话原则**
```

## 示例

### 场景：讨论产品功能权限设计

**对话：**
```
用户：我们在考虑新功能的权限设计，是做成高级会员专享，还是全部用户都能用？

Claude：[讨论 pros/cons...]

用户：好的，那我们先让普通用户也能用基础版本，高级功能留给会员。
帮我记录一下这个决策。
```

**生成的文件：** `2026-01-05-feature-access-freemium-vs-premium.md`

```markdown
# 功能权限：免费增值 vs 会员专享

## 背景
新功能上线前需要确定权限策略，平衡用户获取与商业变现

## 冲突点
**用户增长** vs **会员转化率**

- 全部免费：降低门槛，快速获取用户，但影响付费动力
- 全部付费：提高会员价值感，但限制功能传播

## 具体例子
某社区产品新增 AI 问答功能

- 方案 A（全部免费）：所有用户无限制使用 → 用户增长快，但会员转化率下降 30%
- 方案 B（全部付费）：仅会员可用 → 会员续费率提升，但新用户体验不到价值
- 方案 C（免费增值）：基础版免费，高级功能付费 → 平衡增长与变现

## 我的决策
采用方案 C：基础版对所有用户开放，高级功能（高频使用、高级模型）作为会员权益

**理由：**
- 让用户先体验到价值，建立使用习惯
- 自然引导到付费场景（重度用户会碰到免费限制）
- 降低产品传播门槛

## 决策时的担忧
- 担心免费用户占用过多成本（待验证：监控前三个月成本）
- 担心"免费"定位影响品牌高端感（待验证：用户调研反馈）

## 决策原则提炼
> **先让用户体验价值再谈付费，用使用深度而非功能可见性区分权限**
```

## 最佳实践

### 好的决策记录
✅ 标题描述问题类型，不是解决方案
✅ 冲突点明确，权衡清晰
✅ 具体例子有血有肉
✅ 原则可迁移到其他场景

### 避免的问题
❌ 只记录结果，不记录推理过程
❌ 原则过于抽象，无法指导实践
❌ 缺少后续验证计划
❌ 混淆问题和解决方案

```bash
ls claude-workspace/decisions/
```

```bash
grep -r "权限设计" claude-workspace/decisions/
```

### 按时间查看
```bash
ls -lt claude-workspace/decisions/
```

## 进阶用法

### 定期回顾
建议每月回顾 `decisions/` 目录：
- 验证当时的担忧是否成真
- 更新决策结果
- 提炼更通用的原则

### 主题索引
可以在 `decisions/INDEX.md` 中维护主题索引：
```markdown
# 决策记录索引

## 产品设计
- 2026-01-05: 功能权限策略
- 2026-01-10: 用户引导流程

## 技术架构
- 2026-01-08: 数据库选型
```
