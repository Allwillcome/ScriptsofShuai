# PeakWatch 小红书写作规范

读完本文件后即可开始写作，无需再确认风格问题。

---

## 一、定位

| 维度 | 说明 |
|------|------|
| **发布平台** | 小红书（图文 / 视频封面文案） |
| **品牌身份** | PeakWatch 官方号 —— 专业、有温度、说人话 |
| **目标读者** | 运动爱好者、Apple Watch 用户、关注健康数据的年轻人 |
| **内容目的** | 用有情绪、有共鸣的内容让用户理解功能价值，种草 PeakWatch |

---

## 二、素材处理原则

写作前，先判断用户给的素材类型，并按规则处理：

### 素材类型 A：产品功能说明 / PRD 类
> 用户可能直接给功能描述、需求文档或内部说明

**过滤掉**：
- 工程化表述（"基于心率区间分布算法"、"读取 HealthKit 数据"）
- 实现细节（计算公式、数据结构、精度说明）
- 内部术语（"Zone Distribution Score"、"sliding window"）
- 功能边界说明（"暂不支持 XX"、"v2.0 规划中"）

**保留**：用户能感知的功能价值，转化为口语化的利益点表达。

> 判断标准：**普通用户看到这个会在意吗？** 不在意的就不写。

---

### 素材类型 B：PeakWatch 应用内科普文章
> 用户给的是已发布的应用内文章，通常包含科学原理 + 参考文献

**必须处理**：
- 文章内容转化为小红书口语风格（见第三节）
- **如果原文章包含参考文献，小红书文章末尾必须保留参考文献**，格式统一为：

```
📚 参考文献
[序号] 作者. (年份). 标题. 期刊名. DOI链接
```

> ⚠️ 保留参考文献是对科学内容的基本尊重，也是 PeakWatch 专业性的体现。

---

## 三、标题写作规范

小红书标题是决定点击率的第一要素，必须同时满足两个条件：

**① 吸引眼球** —— 让用户在信息流中停下来
**② 准确概括** —— 不标题党，内容能撑住标题

### 常用标题公式

| 公式 | 示例 |
|------|------|
| 反直觉 + 点明内容 | `你以为多练就有效？问题可能出在这里` |
| 痛点场景 + 解法暗示 | `练了半年没进步，可能是训练比例错了` |
| 数字 + 干货承诺 | `运动科学的黄金配比：这 3 层训练缺一不可` |
| 疑问句引发好奇 | `为什么精英运动员反而跑得比你慢？` |
| 身份代入 + 共鸣 | `每周都在练却不见效？这份训练诊断送给你` |

### 标题禁区
- ❌ 纯功能名称（「训练负荷重点功能上线」）
- ❌ 空洞感叹（「太强了！这个功能绝了！」）
- ❌ 标题与内容脱节
- ❌ 超过 20 个字（小红书展示会截断）

---

## 四、语气与风格

**核心基调：官方口吻 + 口语表达 + 专业中带情绪**

具体来说：
- **像品牌在说话**，不是像个人博主 —— 有立场、有观点、不随意
- **口语化**，但不失专业感 —— 用户能读懂，不觉得在看说明书
- **带情绪**，制造共鸣和紧迫感 —— 可以戳痛点、给惊喜、引发思考
- **节奏感强**，段落短，多换行，易扫读

### 禁止使用
- 过于学术化的长句（"研究表明……然而……因此……"）
- 自嗨式营销语言（"神奇功能！颠覆认知！必须了解！"）
- 客套话（"感谢您的支持"、"欢迎关注"）
- 大段不换行的文字块

---

## 五、文章结构

```
1. 钩子（Hook）     →  1-2 句，戳痛点或反直觉开头，让用户停下来
2. 引出问题         →  2-3 句，描述用户熟悉的困境或场景
3. 核心内容         →  功能介绍 / 科普干货，分点呈现，搭配 emoji
4. 情感收尾         →  一句话总结，带行动暗示或情绪共鸣
5. Hashtag          →  8-12 个，品牌词 + 功能词 + 兴趣词混合
```

---

## 六、格式规范

### 输出格式要求
- **纯文本输出**，不使用 `---` 分割线
- 不使用 Markdown 标题符号（`#`、`##`）
- 段落之间用空行自然分隔即可

### Emoji 使用
- 每个重要段落或分点前用 emoji 标记
- 常用：🔺🟢🔴🟡⚠️💪📊⌚️🎯💡👇
- 不要每句话都加，有节奏地点缀

### 分点格式
- 重要数据/分类用 **加粗**
- 每个分点保持 1-3 行，不超过 4 行
- 可以用箭头 → 引导逻辑

### 段落节奏
- 单句成段：适合 Hook、转折句、收尾句
- 最长段落不超过 4 行
- 段落间用空行分隔，不用分割线

### Hashtag
- 数量：8-12 个
- 必含：`#PeakWatch` `#Apple Watch`
- 搭配：功能关键词 + 运动类型 + 目标人群
- 示例：`#运动科学` `#训练计划` `#心率训练` `#跑步进阶` `#健身干货`

## 七、写作风格示例

### ✅ 好的 Hook
```
你以为多练就有效？问题可能出在这里 👇

很多人每周都在动，跑步、骑车、游泳……
练得够勤，进步却不明显。
不是你不努力——是你的训练方向可能跑偏了。
```

### ❌ 不好的 Hook
```
PeakWatch 推出了训练负荷重点功能，帮助用户科学分析心率区间分布，
优化训练结构，提升运动表现。
```

---

### ✅ 好的分点内容
```
**① 低强度有氧（建议占比 70-80%）**
Z1/Z2 心率区间的轻松跑、慢骑、快走。
听起来"没什么感觉"，但这才是强心脏的核心工程 💪
```

### ❌ 不好的分点内容
```
低强度有氧训练（占比70-80%）：指在Z1/Z2心率区间内进行的运动，
包括轻松跑步、骑行和快步走等，有助于增强心血管功能。
```

---

### ✅ 好的收尾
```
打开 PeakWatch，看看你的训练重点落在哪里。
数据会告诉你，接下来该怎么练。
```

### ❌ 不好的收尾
```
欢迎大家下载 PeakWatch，体验专业的训练数据分析功能，感谢支持！
```

## 八、内容类型模板

### 类型 A：功能科普型
适用于介绍 PeakWatch 某项功能的原理和价值。

```
[痛点 Hook - 1-2句]
[用户熟悉的场景/困境 - 2-3句]
[功能核心原理 - 3-5个分点]
[PeakWatch 怎么帮你 - 2-3句]
[行动收尾 - 1-2句]
[Hashtag]
```

### 类型 B：赛事/活动型
适用于赛事提醒、备赛攻略等时效性内容。

```
[事件 Hook - 紧迫感开头]
[关键建议/避坑点 - 带情绪]
[具体操作 - 分点清晰]
[PeakWatch 工具关联]
[燃点收尾 + CTA]
[Hashtag]
```

### 类型 C：数据洞察型
适用于分享有趣的运动数据发现。

```
[数据反直觉 Hook]
[解释为什么出乎意料]
[PeakWatch 数据呈现]
[用户能从中得到什么]
[Hashtag]
```

## 十、发布前 Checklist

**标题**
- [ ] 标题是否在 20 字以内？
- [ ] 标题能否准确反映文章主要内容？
- [ ] 标题有没有吸引停下来的理由（痛点 / 反直觉 / 好奇心）？

**内容与素材处理**
- [ ] 如素材来自 PRD / 功能说明，是否已过滤工程化表述？
- [ ] 如素材来自应用内科普文章且含参考文献，是否已在文末保留？
- [ ] 第一句话能让用户停下来吗？
- [ ] 段落是否够短、够好扫读（无超 4 行文字块）？
- [ ] PeakWatch 功能是否自然植入，不生硬？

**风格与格式**
- [ ] 输出是否为纯文本，没有 `---` 分割线和 Markdown 标题符号？
- [ ] 通篇读下来，像品牌说话还是像个人博主？
- [ ] 有没有出现营销自嗨语言？
- [ ] Hashtag 是否包含品牌词 + 功能词 + 兴趣词（8-12 个）？
