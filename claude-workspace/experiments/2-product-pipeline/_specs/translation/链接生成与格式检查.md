# Multilingual Documentation Helper

This skill assists with managing a multilingual documentation project, specifically for the "Garmin PeakWatch" project mapping.

## 1. File Organization & Naming
- **Directory Structure**: Store files in language-specific folders:
  - `en/` (English)
  - `zh_cn/` (Simplified Chinese)
  - `zh_tw/` (Traditional Chinese)
  - `ja/` (Japanese)
  - `kr/` (Korean)
  - `de_de/` (German)
  - `es/` (Spanish)
- **Subdirectories**: Group related files (e.g., `weight/`) inside the language folder.
- **Naming Convention**:
  - Use kebab-case (e.g., `fat-free-mass.md`).
  - **Remove language suffixes** (e.g., change `bmi-cn.md` to `bmi.md` inside `zh_cn/`).

## 2. Link Generation Rules
- **Format**: Pure text paths ending in `.html`.
  - Root files: `/filename.html`
  - Subfolder files: `/folder_name/filename.html` (e.g., `/weight/weight.html`)
- **Table Output**: When requested, generate a Markdown table with these columns:
  - `封面下标题` (Title)
  - `文档存放文件夹` (Folder Name)
  - `链接` (Link)

## 3. Markdown Validation
- **Check**: Scan files for syntax errors (broken links, malformed tables, unclosed tags, heading levels).
- **Fix**: Auto-correct issues where possible.
- **Report**: List found issues and applied fixes.

## Usage
- When the user asks to "organize files" or "generate links", follow the rules above strictly.
- If files are in a flat list with suffixes (like `_cn`, `-en`), move and rename them to the structured format.
