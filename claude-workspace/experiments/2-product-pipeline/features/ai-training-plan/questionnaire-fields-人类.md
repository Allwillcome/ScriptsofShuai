```Markdown
- 课程时长：[X 分钟]
- 课程难度：[简单/适中/难，默认根据运动水平推荐]

### 约束条件
- 训练场地：[居家/健身房/户外]
- 可用器械：[根据场地和训练类型列出]
- 伤病情况：[无伤病 / 具体部位]

### 用户偏好
- 希望避免的训练类型：[如有]
- 希望避免的动作：[如有]

### 身体状态（可选）
- 今日训练准备度 / 睡眠质量 / HRV / 静息心率：[数值或趋势，缺失则按历史均值估算]

## 训练课结构要求

### 1. 热身阶段（5-10 分钟）
- 类型：动态热身，先全身激活 → 再针对目标肌群
- 要求：针对本次训练类型和目标肌群，姿势切换流畅

### 2. 训练主体

根据训练类型，参照《训练主体设计规范》（training-body-design.md）执行对应设计逻辑。

### 3. 拉伸放松阶段（5-10 分钟）
- 类型：静态拉伸
- 要求：针对本次训练的目标肌群

## 设计原则

1. **专项性** — 热身和拉伸要匹配训练内容
2. **个性化** — 根据用户目标和训练水平确定强度
3. **流畅性** — 动作与动作之间的姿势、设备切换要平滑，用户体验优先
4. **安全性** — 有伤病部位自动排除相关动作并提供替代方案
```

```Markdown
## 计划设计原则

### 1. 遵循 ACSM-FITTVP 框架
- **F (Frequency)**：每周训练天数和每次训练频次
- **I (Intensity)**：根据训练水平设定强度（%1RM 或 RPE）
- **T (Time)**：每次训练时长
- **T (Type)**：训练类型组合（力量/有氧/灵活性/功能性）
- **V (Volume)**：每周总容量，渐进 overload
- **P (Progression)**：每两周或每周期设定进阶策略

### 2. 波动周期训练 (Daily Undulating Periodization)
- 同一周内安排不同训练类型：
- 力量日（Strength）：大重量、低次数
- 肌肥大日（Hypertrophy）：中等重量、中等次数
- 代谢/耐力日（Endurance）：轻重量、高次数
- 力量训练频率根据目标和训练类型决定：
- 减脂目标：有氧和力量训练交替执行
- 跑步/耐力目标：跑步和力量训练交替
- Hybrid 训练：功能性训练 + 辅助力量

### 3. 训练类型专项设计

根据用户的训练目标，训练计划应包含以下训练类型组合：

#### 减脂目标
- 训练类型：有氧训练、力量训练、HIIT、灵活性训练
- 组合逻辑：力量训练与有氧训练交替安排，高强度与低强度日交替

#### 增肌/力量目标
- 训练类型：抗阻训练、辅助力量训练、灵活性训练
- 组合逻辑：大肌群训练与中小肌群训练交替，复合动作与孤立动作结合

#### 跑步/耐力目标
- 训练类型：跑步训练、力量训练、灵活性训练
- 组合逻辑：跑步日与力量日分开，力量训练针对跑步专项肌群（臀、腿、核心）
- 跑步训练遵循丹尼尔斯跑法周期化                                                             
   
#### Hybrid 训练 (Hyrox/DEKAFIT 等)                                                          
- 训练类型：功能性训练、辅助力量训练、有氧基础训练
- 组合逻辑：Hybrid 日（功能性动作 + 跑步交替）、辅助日（弱项肌群强化）、有氧日（基础耐力）

#### 综合体能提升
- 训练类型：抗阻训练、有氧训练、功能性训练、灵活性训练
- 组合逻辑：根据用户偏好和弱项动态调整四大类型的比例

**具体训练频率和每次时长根据用户的可用时间、训练水平和目标由你综合判断后设计。**

### 4. 周期化安排

根据训练周期时长，划分为以下阶段，每一期至少包含一周

- **基础期**：建立动作模式和基础容量，强度适中
- **提升期**：逐步增加强度和容量
- **减量期**：减少容量，维持强度，为巅峰期做准备
- **巅峰期**：峰值强度和竞技状态

**根据训练周期长度灵活调整各阶段时长，确保训练之间有足够恢复。**
```
