# Intervals.icu Workout Format Documentation

> Complete guide to Intervals.icu plain text workout format for generating training plans

## Overview

Intervals.icu supports a plain text format for creating structured workouts. This format is simple, human-readable, and can be easily generated by AI or scripts.

## Basic Syntax

### Time Formats

| Format | Meaning | Example |
|--------|---------|---------|
| `10m` | 10 minutes | `- 10m 75%` |
| `30s` | 30 seconds | `- 30s 95%` |
| `1m30` | 1 minute 30 seconds | `- 1m30 85%` |
| `1h` | 1 hour | `- 1h 60%` |
| `5'` | 5 minutes (alternative) | `- 5' 70%` |
| `30"` | 30 seconds (alternative) | `- 30" 100%` |

**Important**: `m` means minutes, NOT meters. For distance, use `km` or `mi`.

### Distance Formats

| Format | Meaning | Example |
|--------|---------|---------|
| `5km` | 5 kilometers | `- 5km Z2` |
| `3mi` | 3 miles | `- 3mi 75% pace` |
| `800m` | 800 meters | `- 800m Z4 Pace` |

### Intensity Specifications

#### Cycling (Power-based)

| Format | Meaning | Example |
|--------|---------|---------|
| `75%` | % of FTP | `- 20m 75%` |
| `200w` | Absolute watts | `- 10m 200w` |
| `85-95%` | Power range | `- 5m 85-95%` |
| `Z2` | Zone 2 (56-75% FTP) | `- 30m Z2` |
| `ramp 50%-75%` | Gradual increase | `- 10m ramp 50%-75%` |

#### Running (Pace-based)

| Format | Meaning | Example |
|--------|---------|---------|
| `80% pace` | % of threshold pace | `- 2km 80% pace` |
| `Z2 Pace` | Zone 2 pace | `- 5km Z2 Pace` |
| `Z4 Pace` | Zone 4 pace (threshold) | `- 1mi Z4 Pace` |

**Note**: For running, higher percentage = faster pace.

#### Heart Rate

| Format | Meaning | Example |
|--------|---------|---------|
| `70% HR` | % of max heart rate | `- 15m 70% HR` |
| `95% LTHR` | % of lactate threshold HR | `- 10m 95% LTHR` |
| `Z2 HR` | Heart rate zone 2 | `- 20m Z2 HR` |

### Cadence (Cycling)

| Format | Meaning | Example |
|--------|---------|---------|
| `90rpm` | Target cadence | `- 10m 75% 90rpm` |
| `70-100rpm` | Cadence range | `- 5m 88% 70-100rpm` |

## Structure

### Workout Components

```
Section Title
- Step 1
- Step 2

Repeat Section 5x
- Step 1
- Step 2
```

### Rules

1. **Section Titles**: Lines without `-` prefix are section titles
2. **Steps**: Lines starting with `-` are workout steps
3. **Repetitions**: Add `3x`, `4x`, etc. after section title
4. **Free Text**: Can add description before duration/intensity

## Training Zones

### Cycling Power Zones (% FTP)

| Zone | Name | % FTP Range | Description |
|------|------|-------------|-------------|
| Z1 | Recovery | < 55% | Very easy, recovery |
| Z2 | Endurance | 56-75% | Comfortable, conversational |
| Z3 | Tempo | 76-90% | Moderately hard, sustained |
| Z4 | Threshold | 91-105% | Hard, at lactate threshold |
| Z5 | VO2Max | 106-120% | Very hard, max aerobic |
| Z6 | Anaerobic | > 120% | Extremely hard, sprints |

### Running Pace Zones (% of Threshold Pace)

| Zone | Name | % Pace Range | Description |
|------|------|--------------|-------------|
| Z1 | Recovery | 70-75% | Very easy jog |
| Z2 | Easy | 76-80% | Conversational pace |
| Z3 | Tempo | 81-88% | Comfortably hard |
| Z4 | Threshold | 89-95% | Hard, sustainable ~1hr |
| Z5 | VO2Max | 96-105% | Very hard, 3-8 min efforts |
| Z6 | Sprint | > 105% | All-out, short efforts |

## Advanced Features

### Ramps (Progressive Intensity)

```intervals
- 10m ramp 50%-75%
- 15m ramp 60%-90% 85rpm
```

### Timed Text Prompts

```intervals
- First prompt 33^ Second at 33s <!> 10m ramp 25-75%
```

### Free Text Descriptions

```intervals
- Easy spin at 10m 60%
- Power intervals 5m 95-105%
- Recovery ride 20m 50-60%
```

## Parsing Rules

1. Keywords are case-insensitive
2. First duration/distance found becomes the step duration
3. Intensity ranges use `-` (e.g., `85-90%`)
4. Multiple metrics can be combined (e.g., `10m 75% 90rpm`)
5. Text before metrics becomes step description

## Limitations & Best Practices

### ⚠️ Known Issues

1. **Mixing Metrics**: Combining HR, Power, and Pace in one workout may cause rendering issues
2. **Time Format**: `1h30` works, but always verify rendering
3. **Validation**: Always preview workouts in Intervals.icu after generation

### ✅ Best Practices

1. **Keep it Simple**: Use one primary metric per workout
2. **Clear Sections**: Use descriptive section titles
3. **Logical Progression**: Warmup → Main Set → Cooldown
4. **Test First**: Preview generated workouts before using
5. **Be Specific**: Include target zones/intensities for clarity

## API Integration

When generating workouts programmatically:

```python
def generate_interval_workout(warmup_min, interval_power, interval_min, recovery_min, sets):
    workout = f"""Warmup
- {warmup_min}m 60%

Main Set {sets}x
- {interval_min}m {interval_power}%
- {recovery_min}m 50%

Cooldown
- 10m 55%
"""
    return workout
```
