## 1. 产品概述 (Product Overview)

### 1.1 背景

Intervals.icu 是一个专业且硬核的耐力运动分析平台，提供了海量的数据图表。然而，对于普通用户而言，解读这些数据（如 Power Curve, Decoupling, HR Zones）存在门槛。

### 1.2 目标

开发一款 Chrome 浏览器插件，作为 Intervals.icu 的“AI 伴侣”。它能自动抓取当前页面的运动数据，通过大语言模型（LLM）生成通俗易懂、语气亲切且专业的分析建议，帮助用户理解训练效果。

### 1.3 核心价值

- **降门槛**：将冷冰冰的数据转化为自然语言。
- **个性化**：支持自定义提示词和 AI 模型，满足不同用户的偏好。
- **原生体验**：嵌入式 UI，不打断用户浏览流程。

## 2. 用户流程 (User Flow)

1. **安装与配置**：用户安装插件 -> 首次点击图标/设置 -> 打开独立配置页 -> 选择 AI 厂商（如 OpenAI/API2D） -> 输入 API Key -> 保存。
2. **触发分析**：用户进入 Intervals.icu 的某次活动详情页 (Activity Page) -> 页面右侧显示悬浮球 -> 点击悬浮球展开卡片 -> 点击“开始分析”。
3. **查看结果**：插件显示“思考中” -> AI 流式/一次性返回结果 -> 结果以 Markdown 格式渲染在卡片中。
4. **异常处理**：若 Key 无效或网络错误，卡片内直接提示错误，并提供跳转设置页的入口。

## 3. 功能需求 (Functional Requirements)

### 3.1 前端 UI 交互 (Content Script)

#### 3.1.1 悬浮球 (Floating Widget)

- **位置**：页面右侧中上部，固定定位 (Fixed)。
- **默认状态**：收起状态，仅显示图标（如⚡️）和标题，占用极小空间，避免遮挡网页内容。
- **交互**：点击顶部栏可 展开/收起 面板。
- **样式**：风格需与 Intervals.icu 保持协调，支持简单的动画过渡。

#### 3.1.2 分析面板 (Expanded Card)

- **操作区**：包含“开始分析 (Analyze)”按钮。
- **结果展示区**：
  - 支持 **Markdown 渲染**（加粗、列表、标题、换行）。
  - 分析过程中显示 Loading 状态（文案如：“AI 正在思考...”）。
- **设置入口**：底部提供小文字链接“⚙️ 设置 API”，点击后通过 `runtime` 消息打开设置页。
- **错误提示**：若不在运动详情页（如在首页），点击分析应提示“请进入具体的运动详情页面”。

#### 3.1.3 数据抓取 (Data Scraping)

- **触发时机**：点击“开始分析”按钮时触发。
- **抓取范围**：
  - 当前页面的活动标题 (`h1` 等)。
  - 关键摘要数据 (Summary Text)。
  - 备注/描述 (Description)。
  - *注：暂不通过拦截 API 获取 JSON，以降低开发难度和风险，优先采用 DOM 文本提取。*

### 3.2 后台逻辑 (Background Service Worker)

#### 3.2.1 API 请求代理

- **功能**：接收前端传来的数据，向 AI 服务商发起 HTTP POST 请求。
- **原因**：解决 Content Script 直接请求可能遇到的 CORS（跨域）问题。
- **鉴权**：请求头需包含 `Authorization: Bearer {API_KEY}`。

#### 3.2.2 提示词管理 (Prompt Management)

- **加载逻辑**：
  1. 优先读取用户在“设置页”自定义的 Prompt。
  2. 若无自定义，则读取插件内置的 `prompt.txt` 文件。
  3. 若文件读取失败，使用代码内硬编码的保底 Prompt。
- **变量替换**：支持 `{{LANG}}` 占位符，自动替换为用户浏览器的当前语言（如 `zh-CN` 或 `en-US`）。

#### 3.2.3 插件图标行为

- **急救功能**：当用户点击浏览器地址栏右侧的扩展图标 (`action`) 时：
  - 检测当前是否在 Intervals 页面。
  - 如果是，尝试**强制注入** CSS 和 JS（修复 UI 不显示的问题）。
  - 如果不是，打开设置页。

### 3.3 设置页面 (Options Page)

#### 3.3.1 独立页面

- 使用 `options_ui` 配置，在独立标签页中打开，提供宽敞的配置空间。

#### 3.3.2 AI 厂商预设 (Provider Presets)

- 提供下拉菜单选择常见厂商：
  - **OpenAI** (官方)
  - **DeepSeek** (深度求索)
  - **API2D** (国内代理)
  - **Custom** (自定义)
- **联动效果**：选择厂商后，自动填充对应的 `Base URL` 和 `Model Name` 默认值。
- **获取 Key 引导**：选择厂商后，显示对应的“获取 Key”超链接，直达厂商控制台。

#### 3.3.3 参数配置

- **API Base URL**：允许用户修改（适配各种代理）。
- **API Key**：密码框输入，本地存储。
- **Model Name**：如 `gpt-4o-mini`, `deepseek-chat`。
- **Custom Prompt**：大文本框，允许用户覆盖默认提示词。

#### 3.3.4 数据持久化

- 使用 `chrome.storage.sync` 保存所有配置，支持账号同步。

## 4. 提示词工程 (Prompt Engineering)

### 4.1 角色设定 (Persona)

- **身份**：运动健康专家 + 很懂用户训练习惯的朋友。
- **语气**：自然、口语化、鼓励性、有画面感。

### 4.2 输出规范

1. **直观感受**：一句话总结，带温度，有数据支撑。
2. **关键短评**：2-4段，结构为“观察 + 数据支撑 + 轻建议”。
3. **数据旁白**：轻轻带过步频、触地时间等客观数据，不做过度解读。
4. **格式**：Markdown 格式，禁止使用长篇大论的报告体。

## 5. 技术架构 (Technical Architecture)

### 5.1 文件结构

```
IntervalsAI/
├── manifest.json       # 核心配置 (V3, 权限, Action, Options)
├── content.js          # UI 渲染, DOM 操作, 消息发送
├── styles.css          # 悬浮球与卡片样式 (Scoped CSS)
├── background.js       # API 请求中转, Prompt 读取, 错误处理
├── options.html        # 设置页结构
├── options.js          # 设置页逻辑
├── prompt.txt          # 默认提示词模板
└── icon.png            # 资源文件
```

### 5.2 权限清单 (Manifest Permissions)

- `activeTab`: 获取当前标签页信息。
- `scripting`: 用于点击图标时的强制注入脚本。
- `storage`: 保存 API Key 等配置。
- `host_permissions`:
  - `https://intervals.icu/*` (目标网站)
  - `*://*/*` (允许访问任意 AI 厂商的 API 接口)

### 5.3 错误处理机制

- **网络错误**：捕获 `fetch` 异常，返回具体 HTTP 状态码和错误信息。
- **配置错误**：若未填 Key，前端直接引导跳转设置页。
- **资源错误**：`prompt.txt` 读取失败时，降级使用内置字符串。
- **正则错误**：确保代码中 `replace` 方法使用正确的正则语法。

## 6. 后续迭代计划 (Roadmap)

- **V2.1**：支持流式输出 (Streaming)，让体验更像 ChatGPT。
- **V2.2**：支持一键截图/分享分析结果。
- **V2.3**：增加“历史记录”功能，在本地保存过去的分析。
